# Scope Lock магистерской выпускной квалификационной работы

**Тема:** «Исследование устойчивости нейросетевых архитектур семантической сегментации дорожных сцен к изменениям условий съёмки»  
**Статус:** окончательно зафиксированный объём. Расширение состава данных, моделей, метрик и экспериментов без пересмотра документа не допускается.

## 1. Цель исследования

Разработать воспроизводимый экспериментальный протокол и провести сравнительное исследование устойчивости U-Net, DeepLabV3+ и PSPNet с энкодером ResNet34 к контролируемым изменениям условий съёмки дорожных сцен, а также определить, снижает ли повторное обучение лучшей модели с robust augmentation деградацию качества семантической сегментации.

## 2. Объект и предмет исследования

**Объект** — нейросетевые архитектуры семантической сегментации изображений дорожных сцен.

**Предмет** — изменение интегрального и поклассового качества U-Net, DeepLabV3+ и PSPNet под воздействием синтетических искажений различной тяжести и влияние robust augmentation на устойчивость лучшей модели.

## 3. Задачи исследования

1. Реализовать корректную подготовку Cityscapes: 19 `trainId`-классов, `ignore_index=255`, размер 384×192, nearest neighbour для масок.
2. Реализовать единый протокол обучения U-Net, DeepLabV3+ и PSPNet с ResNet34/ImageNet: восемь эпох, `seed=1` и одинаковые условия сравнения.
3. Оценить три модели на чистых изображениях по mIoU и IoU каждого класса.
4. Сформировать детерминированный benchmark из восьми искажений по трём уровням и оценить деградацию моделей.
5. Выбрать лучшую модель по заранее установленным критериям, повторно обучить только её с robust augmentation и сравнить с базовым вариантом.
6. Выполнить поклассовый анализ и сформировать воспроизводимые результаты в CSV, JSON, PNG, checkpoint и локальном MLflow.

## 4. Проверяемые гипотезы

**H1.** Ранжирование моделей по clean mIoU может отличаться от ранжирования по среднему corrupted mIoU; поэтому clean evaluation недостаточна для оценки устойчивости.

**H2.** Искажения неодинаково влияют на классы Cityscapes; малые, редкие и пространственно тонкие объекты могут демонстрировать более сильное снижение IoU. Вывод принимается только по фактическим результатам.

**H3.** Robust augmentation повышает средний corrupted mIoU и/или worst-case mIoU лучшей модели при снижении clean mIoU не более чем на 1,0 процентный пункт.

## 5. Обязательная экспериментальная программа

### 5.1. Данные и предобработка

Используется только Cityscapes. Из официальной части `train` один раз создаётся внутреннее train/validation-разбиение с `seed=1`. Официальные 500 изображений `validation` применяются только для финальной оценки после выбора модели и фиксации robust augmentation. Официальный `test` не используется.

Разметка содержит 19 стандартных значений `trainId` 0–18 и 255 для игнорируемых пикселей. Изображения приводятся к 384×192 с сохранением отношения сторон 2:1; маски масштабируются только nearest neighbour. Применяется нормализация ImageNet.

### 5.2. Модели и обучение

Реализуются исключительно:

- U-Net + ResNet34;
- DeepLabV3+ + ResNet34;
- PSPNet + ResNet34.

Все модели используют веса ImageNet, `seed=1`, восемь эпох, одно разбиение, одинаковые функцию потерь, оптимизатор и критерий checkpoint. Лучший checkpoint определяется по mIoU внутренней validation-выборки.

### 5.3. Эксперименты

1. Проверка пар «изображение–маска», значений разметки, разбиения и матрицы ошибок 19×19.
2. Обучение трёх базовых моделей на чистых данных.
3. Clean evaluation трёх моделей.
4. Corruption evaluation: восемь искажений × три уровня для каждой модели.
5. Выбор одной лучшей модели.
6. Повторное обучение лучшей модели с robust augmentation при `seed=1` и восьми эпохах.
7. Повторная clean- и corruption-оценка robust-варианта.
8. Поклассовый анализ базовых моделей и двух вариантов лучшей модели.

Фиксируются восемь искажений: снижение яркости, снижение контраста, гауссово размытие, размытие в движении, гауссов шум, JPEG-сжатие, туман и дождь. Каждое имеет три неизменяемых уровня тяжести. Искажения применяются только к изображению; маска не изменяется. Случайные параметры детерминируются идентификатором изображения, видом искажения, уровнем и `seed=1`.

Robust augmentation представляет собой вероятностное применение преобразований из данного набора к обучающим изображениям. Состав, вероятности и диапазоны параметров фиксируются в конфигурации до повторного обучения и не корректируются по официальной validation-выборке.

## 6. Сознательно исключено из реализации

Не реализуются:

- SegFormer и любые дополнительные модели;
- ACDC и иные внешние датасеты;
- diffusion- и foundation-модели, генеративное изменение домена;
- domain adaptation, domain generalization и test-time adaptation;
- ECE, NLL, temperature scaling, CD, rCD, AUC и Safe mIoU;
- дополнительные seed, сложные статистические критерии и усреднение по запускам;
- поиск гиперпараметров, ансамбли, дополнительные функции потерь и более восьми эпох;
- дополнительные искажения, уровни тяжести и seen/unseen-анализ;
- разработка новой архитектуры или заявление о принципиально новом алгоритме.

Эти направления допустимы только в литературном обзоре или перечне дальнейших исследований.

## 7. Основные метрики

Метрики рассчитываются по общей матрице ошибок всего набора без пикселей `ignore_index=255`:

- **IoU класса** — отношение TP к объединению истинной разметки и предсказания класса;
- **mIoU** — невзвешенное среднее IoU по 19 классам;
- **corrupted mIoU** — mIoU для конкретного искажения и уровня;
- **средний corrupted mIoU** — среднее по всем 24 искажённым условиям;
- **ΔmIoU** = `clean mIoU − corrupted mIoU`;
- **retention** = `corrupted mIoU / clean mIoU`;
- **worst-case mIoU** — минимальное mIoU среди 24 условий;
- **peak GPU memory** — максимальная выделенная GPU-память при одинаковом режиме измерения.

## 8. Критерии выбора лучшей модели

Выбор производится на внутренней validation-выборке до финальной оценки. Критерии применяются последовательно:

1. максимальный средний corrupted mIoU;
2. максимальный clean mIoU;
3. максимальный worst-case mIoU;
4. минимальная peak GPU memory при близких результатах.

Результаты считаются близкими, если различие по текущему критерию не превышает 0,5 процентного пункта mIoU. Взвешенный интегральный score не используется.

## 9. Ограничения исследования

Исследование ограничено одним датасетом, тремя CNN-архитектурами с общим энкодером, одним seed, восемью эпохами, разрешением 384×192 и синтетическими искажениями. Один запуск не позволяет оценить межзапусковую дисперсию. Уменьшение разрешения может сильнее влиять на малые классы. Синтетические туман и дождь не полностью воспроизводят реальные физические условия. Выводы относятся только к зафиксированному протоколу и не обобщаются автоматически на другие данные и модели.

4. сформированы CSV, JSON, PNG и локальные MLflow-записи;
